## Build & Run

### iOS App
```bash
cd HealthyPlant
xcodegen generate                # generates .xcodeproj from project.yml
open HealthyPlant.xcodeproj      # build with Cmd+R
```
SPM dependencies (Lottie, Firebase) resolve automatically on first open. If not: File > Packages > Resolve Package Versions.

Requires `GoogleService-Info.plist` in `HealthyPlant/HealthyPlant/Resources/` (not committed).

**Install directly on device:**
```bash
# Get device ID (use xcrun xctrace, NOT xcrun devicectl ‚Äî IDs differ)
xcrun xctrace list devices

xcodebuild \
  -project HealthyPlant/HealthyPlant.xcodeproj \
  -scheme HealthyPlant \
  -destination "id=<DEVICE_ID>" \
  -allowProvisioningUpdates \
  CODE_SIGN_STYLE=Automatic
```

**TestFlight upload (build number must be bumped in project.yml first):**
```bash
cd HealthyPlant
xcodegen generate
xcodebuild archive \
  -scheme HealthyPlant \
  -destination "generic/platform=iOS" \
  -archivePath /tmp/HealthyPlant.xcarchive \
  -allowProvisioningUpdates CODE_SIGN_STYLE=Automatic
xcodebuild -exportArchive \
  -archivePath /tmp/HealthyPlant.xcarchive \
  -exportPath /tmp/HealthyPlant_export \
  -exportOptionsPlist HealthyPlant/ExportOptions.plist
xcrun altool --upload-app \
  -f /tmp/HealthyPlant_export/HealthyPlant.ipa \
  --type ios \
  --apiKey <KEY_ID> --apiIssuer <ISSUER_ID>
```
App Store Connect API key credentials are stored in memory. `ExportOptions.plist` uses `destination: export` (not `upload`) so it works without an Xcode account session.

### Backend
```bash
cd backend/api
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8080
```
Requires `.env` with `ANTHROPIC_API_KEY`, `GCP_PROJECT_ID`, `GCS_BUCKET_NAME`, `GOOGLE_APPLICATION_CREDENTIALS`.

**Deploy manually** (requires `gcloud` auth and Docker):
```bash
GCP_PROJECT_ID=healthyplant-209678 bash backend/scripts/deploy_api.sh
# ENVIRONMENT defaults to "prod" ‚Üí deploys to healthyplant-api-prod
```

Production deploys automatically via GitHub Actions on push to `main` (tests ‚Üí deploy). Dev deploys via `dev_deploy.yml` on PRs. GitHub requires one secret: `GCP_SA_KEY` (service account JSON with Cloud Run + GCR permissions).

## Architecture

**Two-tier app**: SwiftUI iOS client + FastAPI Python backend on Cloud Run, with Firestore (NoSQL) and GCS for storage.

### iOS (MVVM, Swift 5.9)
- **App/** ‚Äî `HealthyPlantApp` (Firebase init, AppDelegate for FCM/APNS), `ContentView` (tab bar + auth/onboarding gate), `AppState`, `AppEnvironment` (prod/dev URL switching via UserDefaults)
- **Models/** ‚Äî `Codable` structs matching backend camelCase JSON
- **ViewModels/** ‚Äî `@MainActor ObservableObject`, one per feature. Call services only, never `APIClient` directly.
- **Views/** ‚Äî Feature folders: Home, Profiles, Garden, Community, Calendar, Assistant, Settings, Onboarding, Sensors, Scan
- **Services/** ‚Äî `APIClient` (base HTTP + SSE), feature services, `AuthService` (Firebase anonymous + Google Sign-In), `ImageUploadService`, `NotificationService`
- **Utilities/** ‚Äî `Theme` (dark theme, green accent `#00C853`), `ViewModifiers` (`.greenOutline()`), date/number extensions

Project generated by XcodeGen from `project.yml`. Bundle ID: `com.healthyplant.app`. Build number: `CURRENT_PROJECT_VERSION` in `project.yml`.

### Tab Structure
5 tabs (custom `TabBarView`): **Home** ¬∑ **Profiles** ¬∑ **Community** ¬∑ **Calendar** ¬∑ **Assistant**

- **Profiles tab** cycles three view modes: booklet ‚Üí grid ‚Üí garden (toggled by toolbar icon)
- **Community tab** was formerly the Garden tab ‚Äî `GardenView.swift` now contains `CommunityTabView`

### Backend (FastAPI, Python 3.12)
- **main.py** ‚Äî CORS, auth middleware, router registration. Auth middleware checks `Authorization: Bearer <Firebase JWT>` first, falls back to `X-User-ID` header (anonymous users).
- **routers/** ‚Äî `/api/v1/` prefix: `profiles`, `garden`, `calendar`, `chat`, `community`, `gardeners`, `sensors`, `auth`, `photos`
- **services/** ‚Äî `FirestoreService` (all DB ops), `AIService` (Claude claude-sonnet-4-20250514 via Anthropic SDK), `StorageService` (GCS signed URLs), `FCMService` (push notifications via `firebase_admin`)
- **models/** ‚Äî Pydantic v2 with camelCase `Field(alias=...)`. All response models use `model_config = {"populate_by_name": True}`.

### Auth & Identity
- Anonymous users: UUID generated on first launch, stored in `UserDefaults`, sent as `X-User-ID` header
- Google Sign-In: Firebase Auth JWT sent as `Authorization: Bearer` header; middleware sets `request.state.user_id` to Firebase UID
- **Account migration**: When anonymous user links Google, iOS calls `POST /api/v1/auth/migrate` with the old anonymous UUID. Backend batch-copies profiles, calendar events, chat messages, sensors, and the gardener profile root doc to the new Firebase UID.

### Firestore Schema
```
users/{userId}                        # gardener profile root doc (bio, experienceLevel, avatarURL, isPublic, fcmToken)
users/{userId}/profiles/              # plant profiles
users/{userId}/calendar_events/
users/{userId}/chat_messages/
users/{userId}/sensors/
users/{userId}/following/{targetId}   # follow graph
users/{userId}/followers/{followerId}
community_plants/{id}                 # denormalised shared plants (sourceUserId, gardenerAvatarURL, ...)
```

### Key Data Flows
- **Profile creation**: POST returns immediately; AI recommendations (sun/water/harvest) generated async via `BackgroundTasks` and written back to Firestore
- **Chat**: SSE streaming ‚Äî user message saved, history + all plant profiles loaded as context, Claude streams chunks, final message saved on `done` event
- **Calendar**: AI generates 7-day care schedule from all profiles; events deduplicated before insert
- **Gardener follow + notify**: Follow writes symmetric docs in both subcollections. On community plant share, `asyncio.create_task` fires FCM multicast to all followers' tokens via `send_each_for_multicast` (wrapped in `run_in_executor`).

### Gardening Rank System
Computed client-side in `GardeningRank.compute(profiles:)` from `sum(ageDays) + count √ó 30`. Tiers: Seedling (0) ‚Üí Sprout (50) ‚Üí Grower (200) ‚Üí Gardener (500) ‚Üí Green Thumb (1000) ‚Üí Expert (2000) ‚Üí Master Gardener (4000). Displayed in Profiles tab toolbar.

### Onboarding Gate (ContentView)
```
App launch
  ‚îî‚îÄ isSignedIn? no ‚Üí SignInPromptView / anon continue
  ‚îî‚îÄ gardenerSetupComplete? (UserDefaults "hp_gardener_setup_complete") no ‚Üí GardenerProfileSetupView
  ‚îî‚îÄ main TabBarView
```
FCM deep link (`"OpenCommunityTab"` NotificationCenter post) switches to the Community tab from anywhere.

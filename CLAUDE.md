This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build & Run

### iOS App
```bash
cd HealthyPlant
xcodegen generate                # generates .xcodeproj from project.yml
open HealthyPlant.xcodeproj      # build with Cmd+R on iPhone 15 simulator (iOS 17+)
```
SPM dependencies (Lottie, Firebase) resolve automatically on first open. If not: File > Packages > Resolve Package Versions.

Requires `GoogleService-Info.plist` in `HealthyPlant/HealthyPlant/Resources/` (not committed).

### Backend
```bash
cd backend/api
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8080
```
Requires `.env` with `ANTHROPIC_API_KEY`, `GCP_PROJECT_ID`, `GCS_BUCKET_NAME`, `GOOGLE_APPLICATION_CREDENTIALS`.

### Tests
```bash
# iOS (from HealthyPlant/)
xcodebuild test -scheme HealthyPlant -destination 'platform=iOS Simulator,name=iPhone 15'

# Backend (from backend/api/)
pytest -v --tb=short
```

### Infrastructure
```bash
cd backend/terraform
terraform init && terraform apply
```

## Architecture

**Two-tier app**: SwiftUI iOS client + FastAPI Python backend on Cloud Run, with Firestore (NoSQL) and GCS for storage.

### iOS (MVVM, Swift 5.9)
- **App/** -- Entry point (`HealthyPlantApp`), `ContentView` with custom 5-tab `TabBarView`, `AppState` environment object
- **Models/** -- `Codable` structs: `PlantProfile`, `CalendarEvent`, `ChatMessage`, `UserSettings`
- **ViewModels/** -- One per feature, `@MainActor` with `@Published` state. Call services, never API directly
- **Views/** -- Feature folders (Home, Profiles, Garden, Calendar, Assistant, Settings) + shared Components
- **Services/** -- `APIClient` (base HTTP + SSE), `PlantService`, `CalendarService`, `ChatService`, `AuthService`, `ImageUploadService`, `NotificationService`
- **Utilities/** -- `Theme` (dark theme, green accent #00C853), date/number extensions

Project generated by XcodeGen from `project.yml`. Bundle ID: `com.healthyplant.app`.

### Backend (FastAPI, Python 3.12)
- **main.py** -- App setup, CORS, auth middleware (X-User-ID header in dev, Firebase JWT in prod), router registration
- **routers/** -- `/api/v1/profiles`, `/api/v1/garden`, `/api/v1/calendar`, `/api/v1/chat` (SSE streaming)
- **services/** -- `FirestoreService` (user-scoped subcollections), `AIService` (Claude claude-sonnet-4-20250514), `StorageService` (signed upload URLs)
- **models/** -- Pydantic request/response models with camelCase aliases matching iOS

### Key Data Flows
- **Profile creation**: POST returns immediately, AI recommendations (sun/water/harvest) generated async via background task and written back to Firestore
- **Chat**: SSE streaming -- user message saved, history + plant profiles loaded as context, Claude streams response chunks, final message saved with `done` event
- **Calendar**: AI generates 7-day care schedule from all user profiles; events deduplicated before insert

### Auth
iOS generates anonymous user ID stored in UserDefaults, sent as `X-User-ID` header on all requests. Middleware attaches it to `request.state.user_id`.

### Firestore Schema
All data under `users/{userId}/` with subcollections: `profiles/`, `calendar_events/`, `chat_messages/`.
